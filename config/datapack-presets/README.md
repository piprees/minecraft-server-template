# Structure-frequency presets

Three variants of the `structures` override datapack, generated by
`scripts/gen-terrain-presets.py`'s sibling `scripts/gen-structure-presets.py`
from the curated dial list in `mods/.ideas/customising-structures.csv`.
Each override is a whole-file copy of the structure_set JSON from the exact
pinned mod jar with only placement values changed — regenerate whenever a
structure mod pin bumps (the weekly mod-updates PR is the checkpoint).

| Preset | Intent |
| --- | --- |
| `default/` | Sparse & natural: mega-dungeons and dense hostile sets rarer, villages and small settlements ~25% more common, fortified villages and castle sets kept, other overworld landmarks slightly rarer. **Shipped active** as `config/datapacks/structures/`. |
| `dense/` | Structure-dense world: dungeon/loot spacing ×0.7, landmarks ×0.8, villages more common, mineshafts ×2 frequency. |
| `sparse/` | Structure-sparse world: dungeon/loot/landmark frequency ×0.5, settlements/maritime ×0.7 (villages kept), MVS clutter thinned. |

## Consumers: swapping preset

Copy a variant over the platform pack name in your overlay — same-name
overlay packs replace platform packs at deploy time:

```bash
cp -r .stack/current/stack/config/datapack-presets/dense/structures overlay/config/datapacks/structures
git add overlay && git commit -m "feat: dense structures" && git push
```

Deleting `overlay/config/datapacks/structures/` returns you to the platform
default on the next full deploy.

## What is deliberately NOT overridden

- **Towns and Towers + Explorify** — Cristel Lib rewrites their placement at
  runtime from `config/` files; a datapack override would fight it.
- **YUNG's and Moog's sets with custom placement types**
  (`yungsapi:enhanced_random_spread`, `moogs_structures:advanced_random_spread`,
  …) — untested against field changes; the generator skips them.
- **Ultra-rare by-design sets** (`nova_structures:shrine_tower`,
  `philipsruins:rare_ruin`) and Dungeons Reborn (no structure sets).
- **Vanilla sets** other than `minecraft:villages`.

Frequency changes never move existing placements; the villages spacing
change re-rolls that placement grid — intended for new worlds (see the v3
migration guide).

## Staying in sync with mod updates

The weekly `mod-updates.yml` PR regenerates these presets (and the
jar-baked terrain presets + theme map) from the freshly re-pinned jars, so
whole-file overrides never drift silently; generator drift warnings appear
in the PR body. Consumer-added structure mods are deliberately NOT covered
here — they keep their mod defaults, and consumers theme them for
per-dimension density via `overlay/config/structure_themes.json` (see the
consumer README).
